# Code of Conduct

MLX TTS Studio should be a practical, respectful project space.

## Expected behavior

- Be considerate and constructive.
- Assume good intent, especially around incomplete bug reports or first contributions.
- Keep technical disagreements focused on the code, docs, models, or user experience.
- Respect privacy. Do not post another person's audio, tokens, logs with personal data, or local file paths without consent.

## Unacceptable behavior

- Harassment, threats, or discriminatory language.
- Personal attacks or repeated disruptive comments.
- Publishing private information or sensitive local data.
- Using the project space to promote unsafe or non-consensual voice cloning.

## Enforcement

Maintainers may edit, hide, or remove comments and contributions that violate this code of conduct. Repeated or serious violations may result in a ban from project spaces.
